1. HTML - The Skeleton

HyperText Markup Language (HTML) is the standard markup language for documents designed to be displayed in a web browser. It gives a webpage its structure, like a skeleton for a body. It uses "tags" like `

` for headings and `

` for paragraphs.

2. CSS - The Style

Cascading Style Sheets (CSS) is what makes a website look good. It's the "clothing" for our HTML skeleton. You use it to add colors, fonts, spacing, and layouts. We use a powerful CSS framework called Tailwind CSS to make things beautiful quickly.

3. JavaScript - The Brains

JavaScript (JS) adds interactivity to a website. If HTML is the skeleton and CSS is the style, JS is the brain and muscles. It makes things happen, like opening a popup when you click a button or running a game. It's the magic behind the scenes!

Static HTML vs. React Apps

Let's break down the two main ways we build for the web.

What's the difference?

Imagine building with blocks. A Static HTML site is like building a model house with basic blocks. It's solid, simple, and every part is pre-defined. You build it once, and it stays that way. This very website is a static site!

A React App is like building with magical, dynamic LEGOs. You create reusable components (like a 'button' or a 'user profile') that can change and update themselves without rebuilding the entire house. It's perfect for complex applications where data changes often, like Facebook or Instagram.

Conceptual comparison: Static sites load faster initially, while React apps are faster at updating content on the page.

Static HTML Site

Structure (File: `index.html`)

Everything is in one file. Simple and straightforward.

<!DOCTYPE html>
<html>
<head>
  <title>My Site</title>
</head>
<body>
  <h1>Hello, World!</h1>
  <p>This is a paragraph.</p>
</body>
</html>

Pros & Cons

  • ‚úîFast & Simple: Very easy to build and loads quickly.
  • ‚úîCheap to Host: Any basic hosting works.
  • ‚ùåHard to Update: For large sites, changing something on every page is a lot of work.
  • ‚ùåNot Interactive: Complex features are difficult to add.

React App

Structure (File: `src/App.js`)

Code is broken into reusable "components". This looks like HTML but it's called JSX.

import React from 'react';

function App() {
  return (
    <div>
      <h1>Hello, React!</h1>
      <p>This is a component.</p>
    </div>
  );
}

export default App;

Pros & Cons

  • ‚úîReusable Components: Build a button once, use it everywhere.
  • ‚úîDynamic & Interactive: Great for complex apps where data changes.
  • ‚ùåMore Complex: Requires setup and a "build" step.
  • ‚ùåSlightly Slower First Load: Needs to load the JavaScript library.

Deployment Guide: From VS Code to Live Site

This is your step-by-step guide for your specific setup: VS Code -> GitHub Desktop -> Namecheap Shared Hosting.

For a Static HTML Site:

  1. In VS Code: Write your `index.html`, CSS, and JS files. Save everything in a folder on your computer.
  2. In GitHub Desktop: Click "Add Existing Repository". Select the folder you created. Write a summary (e.g., "Initial commit") and click "Commit to main". Then click "Publish repository" to send it to GitHub online.
  3. In Namecheap cPanel: Open "File Manager". Navigate to `public_html`. If you're using a subdomain (e.g., `tech.sazi.life`), go into the folder with that name.
  4. Click "Upload" and upload all your files (`index.html`, etc.) one by one. That's it, your site is live!

For a React App (This is the tricky part!):

  1. In VS Code Terminal: You need to "build" the app. Run the command: `npm run build`. This creates a `build` folder with all the optimized static files.
  2. Find the `build` folder: It will be inside your project folder.
  3. In GitHub Desktop: You don't need to push the `build` folder to GitHub, only your source code (`src` folder, `package.json`, etc). This is your backup.
  4. In Namecheap cPanel: Open "File Manager" and go to your `public_html` or subdomain folder.
  5. Click "Upload". Before you upload, ZIP the contents of the `build` folder (NOT the folder itself). Upload this single `.zip` file.
  6. Right-click the `.zip` file in File Manager and select "Extract". This will place all the optimized files in your hosting folder. Your React app is now live!
  7. CRITICAL STEP FOR REACT ROUTER: If your React app uses different pages (routing), you must create a file called `.htaccess` in the same folder and add the code from the "Static vs React" tutorial to make sure refreshing the page doesn't cause a 404 error.
